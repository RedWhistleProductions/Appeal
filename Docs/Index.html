Appeal Documentation

Appeal is a general-purpose command-stream language with a cross-platform runtime, platform-independent modules, and platform-specific plugins.

Project Shape

The current project keeps the Appeal runtime, module source, plugin source, runtime resources, and documentation together while the API is still evolving.

  • Source/ contains the Appeal app and core runtime.
  • Modules/ contains platform-independent module source code.
  • Plugins/ contains platform-specific plugin source code.
  • Resources/Appeal/ contains Appeal scripts such as Main.Appeal and Config.Appeal.
  • Resources/Modules/ contains built module and plugin binaries used at runtime.

Conventions

Each module should eventually have a Test_<Module>.Appeal script and a Test_<Module> plugin. The test plugin exports every function expected by the module and prints function calls to the terminal, making it both a test backend and a template for real plugins.

Every module should include a standard Debug 0 / Debug 1 command. Module wrappers should keep their own debug flag and forward it to a plugin export named Debug(bool Enable) when the plugin provides one.
