Audio Module

The Audio module provides Appeal commands for music, sound effects, playlists, and mute/playback control. The module itself is platform-independent; the loaded plugin provides the actual audio backend.

Runtime Layout

Resources/Modules/Audio/Audio.dll
Resources/Modules/Audio/Test_Audio/Test_Audio.dll
Resources/Modules/Audio/Appeal_SDL2_Audio/Appeal_SDL2_Audio.dll

Loading

The module is usually loaded from Config.Appeal with Add_Dynamic_Module.

Add_Dynamic_Module Audio Test_Audio

The second argument is the plugin name. For example, Test_Audio loads the test plugin, while Appeal_SDL2_Audio loads the SDL2 backend after it has been built and installed.

Plugin Naming

Backend plugins should use Appeal_<BackendLibrary>_<Module>. This keeps Appeal plugin DLLs recognizable while still showing the underlying library or platform backend.

Appeal_SDL2_Audio.dll
Appeal_SDL2_Audio.so

Appeal Commands

Command Arguments Behavior
Audio Init pluginPath Initializes the module with a plugin. This is normally called by Add_Dynamic_Module.
Audio Set_Paths resources Sets the resource root used by the audio plugin.
Audio Load_Play_List playlistFile Loads a playlist from the music resource folder.
Audio Load_Song name musicFile Loads a song and stores it under a script-facing name.
Audio Load_SFX name sfxFile Loads a sound effect and stores it under a script-facing name.
Audio Play none Plays the current track.
Audio Play_Song name Finds and plays a named song.
Audio Play_Track trackNumber Moves to a numbered track and plays it.
Audio Play_SFX none Plays the current sound effect.
Audio Play_SFX_Name name Finds and plays a named sound effect.
Audio Stop none Stops music playback.
Audio Next none Moves to the next track and plays it.
Audio Back none Moves to the previous track and plays it.
Audio Mute none Mutes sound effects and stops music.
Audio UnMute none Enables audio and resumes music.
