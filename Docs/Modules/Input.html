Input Module

The Input module provides platform-independent commands for keyboard, mouse, controller, and action binding systems.

Current Status

Test_Input is available as the template plugin. Appeal_SDL2_Input currently supports keyboard, mouse, and SDL game controller actions, button state queries, mouse position/delta/wheel, named function bindings, and controller axis range bindings.

Resource Layout

Resources/Input/Bindings/
Resources/Input/Profiles/

Intended Responsibilities

Area Purpose
Keyboard Read key state and bind key events to Appeal functions or named actions.
Mouse Read position, motion, wheel, button down, button up, and click state.
Controller Read digital buttons, analog triggers, sticks, and controller connection state.
Control Sets Group bindings into named profiles such as Default, Menu, or Player_1.
Actions Let scripts ask for gameplay or UI actions without depending on a specific device.

SDL2 Backend

Appeal_SDL2_Input uses SDL2 events and state queries. Key names may be written as SDL names such as A, Space, and Escape, or with the Appeal prefix such as Key_A, Key_Space, and Key_Escape.

Mouse buttons currently supported are Left, Middle, Right, X1, and X2.

Controllers use SDL game controller names such as Button_A, Button_Start, DPad_Up, Left_Stick_X, Left_Stick_Y, Right_Stick_X, Right_Stick_Y, Left_Trigger, and Right_Trigger. Prefix controller-specific bindings with the controller number, such as 0:Button_A.

Open_Controller -1 enables automatic controller management. If a controller disconnects and SDL reports it again later, the backend reopens it during the next Input Update. Numbered controller opens are also remembered until the matching Close_Controller call.

Plugin Convention

The Input module follows the same plugin pattern as Audio and Graphics:

Plugins/Input/Test_Input/
Plugins/Input/SDL2_Input/

Test_Input prints every function call to the terminal. Appeal_SDL2_Input is the default cross-platform backend until specialized platform plugins exist.
