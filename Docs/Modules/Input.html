Input Module

The Input module is intended to provide platform-independent commands for keyboard, mouse, controller, and action binding systems. In the current source tree, Modules/Input/src/main.cpp is still an Audio-shaped placeholder, so this page documents the current status and the intended direction.

Current Status

The current Input module source exports the same command names as the Audio module. It should not yet be treated as a finished input API.

Intended Responsibilities

Area Purpose
Keyboard Read key state and bind key events to Appeal functions or named actions.
Mouse Read position, motion, wheel, button down, button up, and click state.
Controller Read digital buttons, analog triggers, sticks, and controller connection state.
Control Sets Group bindings into named profiles such as Default, Menu, or Player_1.
Actions Let scripts ask for gameplay or UI actions without depending on a specific device.

Plugin Convention

The Input module should eventually follow the same plugin pattern as Audio:

Plugins/Input/Test_Input/
Plugins/Input/SDL_Input/

Test_Input should print every function call to the terminal. SDL_Input should be the default cross-platform backend until specialized platform plugins exist.
